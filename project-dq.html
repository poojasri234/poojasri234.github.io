Data Quality & Governance Scorecard

A rules-based scorecard to measure completeness, validity, uniqueness, and consistency so stakeholders can trust reporting outputs.

SQL Python Power BI Documentation

1) Business problem

  • Reporting accuracy suffers when key fields are missing, duplicated, or inconsistent.
  • Teams need measurable data quality monitoring over time, not one-time fixes.
  • Data issues should be visible with clear ownership and rule definitions.

2) DQ dimensions

  • Completeness: missing critical values
  • Validity: invalid formats or impossible values
  • Uniqueness: duplicate keys or duplicate rows
  • Consistency: mismatched categories, standardization issues

3) Dataset (example)

Use any dataset; the “premium” part is the rule design + documentation.

  • Primary key field(s) defined (example: customer_id / ticket_id).
  • Critical fields defined (example: date, status, category, amount).
  • Allowed values documented for category/status fields.

4) Process

Rule library

  • Created rule definitions with clear pass/fail conditions.
  • Wrote SQL checks to output bad_count and bad_percent per rule.
  • Maintained a rule catalog (rule_name, description, threshold, owner).

Scorecard

  • Computed DQ % and an overall DQ score by dataset/domain.
  • Tracked DQ score trend over time.
  • Highlighted top failing fields and likely root causes.
